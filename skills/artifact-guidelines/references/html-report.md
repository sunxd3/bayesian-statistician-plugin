# HTML Report Format

Use HTML (not Markdown) for the final written deliverable of any phase — EDA reports, model critiques, experiment summaries, final synthesis reports. A single self-contained HTML file can embed plots inline, render tables natively, and be opened or shared as a single artifact. Plot PNGs sit alongside the HTML in the same directory and are referenced via relative paths (`<img src="plot_01.png">`). See `markdown-report.md` for when Markdown is still the right choice (logs, README-style notes).

## Design philosophy

The aesthetic target is **editorial/scientific magazine**, not a dashboard. Think: reading a methods paper online, with the figures inline. Restrained, typography-driven, light theme, no chrome.

- **Refined minimalism with substance.** Heavy on typography and whitespace; sparse on color. The report's *content* is the visual richness — figures and tables — not gratuitous design elements.
- **Light theme.** Warm off-white background, deep ink text, one muted accent color. No dark mode toggle.
- **Single column, generous measure.** Reading column ~720px max. Plots and wide tables can break out wider via `figure.wide` blocks.
- **Hierarchy via type, not color.** Section weight comes from font size, spacing, and rule lines — not from background blocks or colored callouts.

## Writing rules

- Lead with the key finding or verdict — first paragraph after the headline.
- Support every claim with evidence: an embedded figure, a statistic, a diagnostic.
- Reference plot files in both `<figcaption>` and `alt` text — readers can find them on disk.
- Short paragraphs of complete sentences. Lists only when they genuinely clarify
  (e.g., model assumptions, validation criteria).
- Document what worked, what didn't, and what you tried — negative results are
  findings.
- Minimal headers and bold. No excessive emphasis. The plots are the visual richness.

## Typography

Use Google Fonts (loaded once in `<head>`):

- Headlines (`h1`, `h2`, section titles): **Fraunces** — variable serif, characterful display weights. Use weight 600–800 for display, mild optical sizing.
- Body (`p`, `li`, table content): **Source Sans 3** — clean humanist sans, weights 400/600.
- Numbers, code, table data: **JetBrains Mono** — tabular figures for alignment.

Pairing rationale: Fraunces gives the page a scholarly, slightly idiosyncratic voice (matches Bayesian/statistical tradition); Source Sans keeps body readable; JetBrains Mono aligns columns in tables and gives numerical results their own visual register.

## Color palette (CSS variables)

```css
:root {
  --bg:        #FAF7F0;  /* warm off-white */
  --bg-soft:   #F1ECDF;  /* secondary surface — table headers, callouts */
  --ink:       #1F2933;  /* primary text — deep ink, not pure black */
  --ink-muted: #5B6470;  /* secondary text, captions, axis labels */
  --rule:      #D8D2C0;  /* hairline rules and table borders */
  --accent:    #2D5F4F;  /* muted forest teal — links, key numbers, headline rule */
  --warning:   #B14D2A;  /* burnt sienna — for flags, exceptions, risks */
}
```

Use color sparingly. Body text is `--ink`; links and key statistical results are `--accent`; flagged risks or exceptions are `--warning`. Avoid more than one accent on screen at a time.

## Page structure

```
<header>
  Headline (h1, Fraunces)
  Dek / subtitle (one sentence, italic Source Sans)
  Metadata strip (input · run date · agent name · plugin version)
</header>

<main>
  <section> Headline summary — top-of-page narrative; what's the key finding?
  <section> [domain-specific sections]
            — each agent decides its section list and declares it in
              its Interface > Side effects
            — every section gets an <h2> with a hairline rule above it
            — figures embedded inline within their relevant section
  <section> Risks and Pitfalls
            — flagged items list using the --warning accent
  <section> Recommendations / Next Steps
            — table or list of actions / encodings / decisions
</main>

<footer>
  Generated by <agent-name> · plugin version · timestamp
</footer>
```

## Visual conventions

- **Section rules.** Each `<h2>` preceded by a hairline rule (`border-top: 1px solid var(--rule)`) and generous top padding. No icons, no badges.
- **First letter of headline summary.** Optionally a drop cap in Fraunces 800.
- **Figures.** `<figure>` with `<img>`, `<figcaption>` in italic Source Sans, `--ink-muted`. Reference plots by their filename in the caption too, so readers can find them on disk.
- **Tables.** Thin borders only between rows, no vertical lines. Header row in `--bg-soft`. Numeric columns right-aligned, tabular figures via JetBrains Mono. Variable names in monospace.
- **Inline statistics.** Render `r = +0.40` or `ICC = 0.43` in JetBrains Mono with `color: var(--accent)`.
- **Hypothesis blocks.** Use `<aside>` or `<div class="hypothesis">` with `background: var(--bg-soft)` and a left border in `--accent`. One per story.

## Embedding plots

```html
<figure>
  <img src="plot_07.png" alt="Short, descriptive alt text">
  <figcaption>
    <strong>F5.</strong> One or two sentences stating the specific observation
    visible in the plot, in plain language. <em>plot_07.png</em>
  </figcaption>
</figure>
```

Wide figures (variance decomposition, multi-panel summaries) get `<figure class="wide">` to break out beyond the reading column.

## Self-contained

Inline the CSS in `<style>` in the document head — no external stylesheets. Google Fonts can be loaded via the standard `<link>` tags. This keeps the report a single file that can be opened or shared without extra assets.

## What not to do

- No JavaScript unless a real interaction is needed (sliders for parameters, for example). Most reports are static.
- No emoji, no icon fonts, no decorative SVG illustrations. The plots ARE the visual interest.
- No tabs or accordions — the report should read top to bottom.
- No dashboard-style summary cards with giant numbers in colored tiles. Numbers belong in prose and tables.
- No dark mode, no theme toggles, no logos.

## Skeleton template

```html
<!doctype html>
<html lang="en">
<head>
  <meta charset="utf-8">
  <meta name="viewport" content="width=device-width, initial-scale=1">
  <title><!-- Report title --></title>
  <link rel="preconnect" href="https://fonts.googleapis.com">
  <link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
  <link href="https://fonts.googleapis.com/css2?family=Fraunces:opsz,wght@9..144,400;9..144,600;9..144,800&family=Source+Sans+3:wght@400;600&family=JetBrains+Mono:wght@400;500&display=swap" rel="stylesheet">
  <style>
    :root {
      --bg:#FAF7F0; --bg-soft:#F1ECDF; --ink:#1F2933; --ink-muted:#5B6470;
      --rule:#D8D2C0; --accent:#2D5F4F; --warning:#B14D2A;
      --serif: 'Fraunces', Georgia, serif;
      --sans:  'Source Sans 3', system-ui, sans-serif;
      --mono:  'JetBrains Mono', ui-monospace, monospace;
    }
    * { box-sizing: border-box; }
    html, body { background: var(--bg); color: var(--ink); }
    body {
      font-family: var(--sans);
      font-size: 17px;
      line-height: 1.6;
      margin: 0;
    }
    main { max-width: 720px; margin: 0 auto; padding: 4rem 1.5rem 6rem; }
    header { max-width: 720px; margin: 0 auto; padding: 4rem 1.5rem 1.5rem; border-bottom: 1px solid var(--rule); }
    h1 {
      font-family: var(--serif);
      font-weight: 800;
      font-size: clamp(2.2rem, 4vw, 3.2rem);
      line-height: 1.1;
      letter-spacing: -0.01em;
      margin: 0 0 0.5rem;
    }
    h2 {
      font-family: var(--serif);
      font-weight: 600;
      font-size: 1.7rem;
      margin: 3.5rem 0 1rem;
      padding-top: 2rem;
      border-top: 1px solid var(--rule);
      letter-spacing: -0.01em;
    }
    h3 { font-family: var(--serif); font-weight: 600; font-size: 1.25rem; margin: 2rem 0 0.5rem; }
    p, li { color: var(--ink); }
    .dek { font-style: italic; color: var(--ink-muted); font-size: 1.1rem; margin: 0 0 1rem; max-width: 60ch; }
    .meta { font-family: var(--mono); font-size: 0.85rem; color: var(--ink-muted); margin-top: 1rem; }
    a { color: var(--accent); text-decoration: underline; text-underline-offset: 2px; }
    code, kbd, samp { font-family: var(--mono); font-size: 0.92em; }
    .stat { font-family: var(--mono); color: var(--accent); font-weight: 500; }
    .warn { color: var(--warning); }

    figure { margin: 2rem 0; }
    figure img { width: 100%; height: auto; display: block; border: 1px solid var(--rule); }
    figcaption { font-style: italic; color: var(--ink-muted); font-size: 0.95rem; margin-top: 0.5rem; max-width: 60ch; }
    figure.wide { max-width: 980px; margin-left: calc(50% - 490px); }

    table { width: 100%; border-collapse: collapse; margin: 1.5rem 0; font-size: 0.92rem; }
    thead th { background: var(--bg-soft); text-align: left; padding: 0.6rem 0.8rem; border-bottom: 1px solid var(--rule); font-weight: 600; }
    tbody td { padding: 0.5rem 0.8rem; border-bottom: 1px solid var(--rule); }
    tbody tr:last-child td { border-bottom: none; }
    td.num, th.num { text-align: right; font-family: var(--mono); }
    td.var { font-family: var(--mono); }

    .hypothesis { background: var(--bg-soft); border-left: 3px solid var(--accent); padding: 1rem 1.25rem; margin: 1.5rem 0; }
    .hypothesis h3 { margin-top: 0; }
    .risk { color: var(--warning); }

    footer { max-width: 720px; margin: 4rem auto 2rem; padding: 1.5rem; border-top: 1px solid var(--rule); font-family: var(--mono); font-size: 0.85rem; color: var(--ink-muted); }

    @media (max-width: 720px) {
      figure.wide { max-width: 100%; margin-left: 0; }
    }
  </style>
</head>
<body>
  <header>
    <h1><!-- Headline --></h1>
    <p class="dek"><!-- one-sentence dek summarizing the headline --></p>
    <p class="meta"><!-- input · run date · agent name · plugin version --></p>
  </header>

  <main>
    <!-- sections go here -->
  </main>

  <footer>
    Generated by <code><!-- agent-name --></code> (bayesian-workflow plugin).
  </footer>
</body>
</html>
```

Use this as a starting point; adapt content to the agent's report. Do not add chrome, decorations, or features beyond what's listed unless the work genuinely requires it.
